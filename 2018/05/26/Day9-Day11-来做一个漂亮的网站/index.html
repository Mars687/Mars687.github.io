总结

尽管程序语言在语法框架内赋予编程人员很大的自由度，也不能信马由缰，按自己的想法随便写。有时面对自己写的层次不清的，结构混乱的代码，真的有种崩溃的感觉。看自己写的东西尚且如此，给别人看情况可能要更加糟糕。所以，无论是对于自己还别人，代码可维护性都极为重要。

HTML

html 代码是网页的实体内容，一份结构清晰的文档将大大减少添加样式的工作量。

个人认为，可以把一份文档看成是二维平面的编码。

  • 纵向看，文档内容按时间顺序排列，依次解析整篇文档。
  • 横向看，文档内容按元素层次排列，通过代码缩进来实现。标签越靠左的元素层次越高，为被包含元素的父元素。

一份书写好的文档，纵向同层次的元素分界明显，形成一个个代码块；横向父子元素层次结构清晰。

CSS

  • CSS 应有条理使用选择器，不要想到什么就临时创建一个选择器（特指class和id），造成选择器滥用。
  • 要尽可能提炼文档中元素表现的相同点，使用公用样式来实现它，样式复用大大减少代码量，也降低了样式发生错误的概率。
  • 每个样式使用时必须明确其作用的元素群体。包含很多元素的大类在应用样式时要格外小心，很容易造成“样式污染”，影响其他样式的正确表现。有点类似 JavaScript 中全局变量污染。

比如，如下 CSS 编码错误，引起样式污染，导致浪费了很长时间才发现问题的根源。

/* 第二部分2nd */
.box-2nd {
    height: 710px;
    text-align: center;
}
.box-2nd .main {
    width: 1040px;
}
.box-2nd .text {
    padding-top: 80px;
    padding-bottom: 30px;
    line-height: 30px;
    font-weight: 600;
    font-size: 14px;
}
.box-2nd .title {
    width: 700px;
    margin: 0 auto;
    margin-bottom: 80px;
    line-height: 1;
}
.box .item {
    float: left;
    position: relative;
    width: 430px;
    height: 165px;
    box-sizing: border-box;
    text-align: left;
    padding-left: 76px;
}
.box .item:nth-child(even) {
    margin-left: 140px;
}
.box-2nd .item-title {
    margin-top: 0;
    margin-bottom: 15px;
    font-size: 20px;
    font-weight: 700;
}

还有要格外注意小心书写错误，比如在引入字体时，正确的代码是：

@font-face {
    font-family: 'Montserrat-ExtraLight';
    src: url('./font/Montserrat-ExtraLight.ttf') format('truetype');
}

误将字体格式 .ttf 写成 .tff。

这些错误耗费了大量的调试时间，一定要牢记！
