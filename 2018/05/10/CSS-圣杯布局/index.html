“圣杯”历史简介

在谈“圣杯布局”前我们先聊一下“圣杯”。

根据维基百科[1]：

圣杯是一个容器，它是亚瑟王文学中的一个重要主题。 不同的传统将它描述为具有神奇力量的杯子、碟子或石头，它能提供无限的幸福，永恒的青春或生命。 术语“圣杯”通常用于表示追求具有重要意义的对象或目标。

根据百度百科[2]：

圣杯（San-greal）是在公元33年，犹太历尼散月十四日，也就是耶稣受难前的逾越节晚餐上，耶稣遣走加略人犹大后和11个门徒所使用的一个葡萄酒杯子。耶稣曾经拿起这个杯子吩咐门徒喝下里面象征他的血的红葡萄酒，借此创立了受难纪念仪式。后来有些人认为这个杯子因为这个特殊的场合而具有某种神奇的能力。相传彼世安温的魔法炉则是圣杯的前身。




最后的晚餐[3]



根据纽约邮报(New York Post)2014年3月31日的一则新闻，西班牙历史学家声称他们发现了圣杯，这是传说中耶稣在最后的晚餐上喝过的杯子。他们解释说，玛瑙圣杯隐藏在另一个古老的容器里，这个容器被称为DoñaUrruca圣杯，圣杯位于莱昂的圣伊西多尔大教堂。历史学家的研究得到了科学约会的支持，估计这个杯子是在公元前200年至公元100年之间制作的。[4]


“圣杯布局”起源

“圣杯布局”则源自 Matthew Levine 2006年1月30日的一篇文章：In Search of the Holy Grail。文章开篇作者也说他并不想夸大这种布局的重要性，但既然大家都这么称呼这种布局，而且也知道它的含义，也只能如此了。

所谓的“圣杯布局”就是一个三栏布局，一栏用于导航的固定宽度侧边栏，另一栏用于放Google Ads或Flickr照片，中间一栏则是整个文档的核心。 这种布局在博客黄金时代具有广泛的适用性。

“圣杯布局”的实现

我们逐步来实现这个“圣杯布局”：

STEP 1:构建框架



为构建上图这样的布局框架我们首先创建页眉(header)、页脚(footer)和容器(container):

<div id="header">Header</div>
<div id="container"></div>
<div id="footer">Footer</div>

然后根据左右两栏的宽度给出container的内边距（这里我们假设左栏200px，右栏150px）

#container {
  padding-left: 200px;   /* LC width */
  padding-right: 150px;  /* RC width */
}

Step 2:添加列

在基础的框架上添加列：

<div id="header">Header</div>
<div id="container">
    <div id="center" class="column">Center</div>
    <div id="left" class="column">Left</div>
    <div id="right" class="column">Right</div>
</div>

给文档增加合适的宽度和浮动，让这几栏处于一行，注意清除浮动栏下方footer的浮动：

#container .column {
    float: left;
}
#center {
    width: 100%;
}
#left {
    width: 200px;  /* LC width */
}
#right {
    width: 150px;  /* RC width */
}
#footer {
    clear: both;
}

请注意，中间栏的100％宽度指的是容器container的宽度，不包括padding。 当布局结合在一起时，我们会再次看到这100％宽度，它仍指container的center宽度。



现在我们想让左右两栏分别置于中间栏的两侧，但由于中间栏占据了100%的可用空间，左右两栏被挤到了中间栏的下方。

Step 3：将左栏拉到它本该占据的位置

为将左栏拖动到正确的位置，首先给它添加负的外边距。针对负外边距简单解释一下。

  • 静态元素的负外边距

负外边距像这样：

#content {margin-left:-100px;}


上图展示了一个static元素使用负边距之后的情况。[5]

将之前的“圣杯布局”例子中的 column 类设定为静态元素，我们给 Left 施加负外边距看一下具体效果。

<!DOCTYPE html>
<html>
<head>
    <title>圣杯布局</title>

    <style>
        body {
            min-width: 550px;      /* 2x LC width + RC width */
            padding:0;
            margin:0;
            text-align: center;
            font-size: 30px;
        }
        #header {
            height: 50px;
            width: 100%;
            background-color: #fa0;
        }
        #container {
            padding-left: 200px;   /* LC width */
            padding-right: 150px;  /* RC width */
        }
        #container .column {
            height: 260px;
            position: static;
        }
        #center {
            width: 100%;
            background: #D6D6D6;
        }
        #left {
            width: 200px;          /* LC width */
            right: 200px;          /* LC width */
            background: #E6D;
            margin-top: -100px;
        }
        #right {
            width: 150px;          /* RC width */
            background: #77BBDD; 
        }
        #footer {
            height: 50px;
            width: 100%;
            background-color: #fa0;
            clear: both;
        }
    </style>

</head>
<body>
<div id="header">Header</div>
<div id="container">
    <div id="center" class="column">Center</div>
    <div id="left" class="column">Left</div>
    <div id="right" class="column">Right</div>
</div>
<div id="footer">Footer</div>
</body>
</html>
  1. 当给一个static元素margin-top或margin-left负外边距时，它把元素向特定的方向拉:



margin-top: -100px



margin-top负外边距常用来使元素垂直居中，详见How do negative margins in CSS work and why is (margin-top:-5 != margin-bottom:5)?



margin-left: -100px




2. 当给一个static元素margin-bottom或margin-right负外边距时，它并不会把元素向下或右拉，相反，它会把后面的元素往里面拉，从而覆盖自己:




margin-bottom: -100px






margin-right: -100px



由于 Left 右侧没有元素，所以margin-right: -100px与原来的布局一样，没有任何变化。

* 浮动元素的外边距

如果对一个浮动元素使用负外边距，它会产生一个空白，其他元素就可以覆盖这一部分。这个技巧可以很好地用于流动布局。




margin-top: -100px






margin-left: -100px






margin-right: -100px






margin-bottom: -100px




随着我们逐步减小 Left 的margin-left中的值（如从margin-left:-100px 到margin-left:-180px），左栏 Left 逐渐向左移动，右栏 Right 紧随其后，直到margin-left:-200px，右栏位于container的左边界，左栏跳至 Center 区域右边界，如下图：




margin-left: -200px



当我们继续减小margin-left中的值，右栏固定不动，左栏继续向左移动。当margin-left: -1216px时，左边栏达到了预期位置，如图所示：




margin-left: -1216px



为什么是1216px呢？看下面的图就明白了：




像素尺寸链



上图有些类似机械加工工艺中的尺寸链[7]，我们姑且引入一个像素尺寸链的概念。这里由于我的显示屏横向分辨率为1366px,所以整体尺寸为1366px。我们之前设定:

> padding-left: 200px; padding-right: 150px;

所以，左右两侧分别为200px,150px。
以上3个尺寸构成整个像素尺寸链的组成环， Center 的宽度自然成为了该像素尺寸链中的封闭环。其宽度为：

WidthCenter = 1366px - 200px - 150px = 1016px

此外，显示器宽度，即 body 元素的宽度为此像素尺寸链的增环（该组成环的变动引起封闭环同向变动，即同时增大或减小）。比如屏幕分辨率由1366px增加到1600px,那么 Center 的宽度同时增加234px变为1250px。
padding-left 和 padding-right 为此像素尺寸链的减环（该类组成环的变动引起封闭环的反向变动）
比如padding-left由200px减小至150px，那么 Center 的宽度同时增加50px变为1066px。

上述给 Left 添加负外边距的过程实际上是跨过来 padding-left + Center 的宽度，即：

200px + 1066px = 1266px

这种方式需要计算增加了一定的任务量，大大降低了效率。但这不是主要的问题，关键在于不同的设备屏幕尺寸不同（即body元素宽度不同），这会影响左栏的最终位置。
其实，我们左栏可以先给 Left 添加一个100%的负外边距，使它穿过中间栏。这里的100%是container中间的宽度，也就是 Center 栏的宽度。此时左栏与中间栏相重叠，它们左边对齐，效果如下：



现在需要把左栏继续向左推，使它达到预期位置。我们采用相对定位 + 左栏宽度补偿的方式来实现它。

#container .column {
    height: 260px; 
    position: relative;
    float: left;
}
#left {
    width: 200px;          /* LC width */
    left: -200px;          /* LC width */
    background: #E6D;
    margin-left: -100%;  
}

此时，左栏达到了预期位置。



注意：相对定位指的是“相对于它自己”,其中top, left分别相对于原来的位置向下、向右移动，而bottom, right分别相对于原来的位置向上、向左移动。



相对定位并没有从常规流（normal flow）中移除该元素，只是通过补偿（offset）的方式使的这个元素在视觉上看起来相对于原来的位置移动到了一个新的位置。实际上，这个元素在常规流中仍占据着原来的位置。

Step 4：将右栏拉到它本该占据的位置
同理，我们将右栏放到右边：

#right {
    width: 150px;          /* RC width */
    margin-right: -150px;  /* RC width */
    background: #77BBDD;  
}


疑点

这里有一个问题margin-right从0–>-150px过程中,右栏始终不动，直到margin-right: -150px，右栏突然移动到上一行最右端（即预期位置）。这有点费解，没有查到解释这个细节的资料，暂且记住这个特性。

至此我们完成了一个简单的“圣杯布局”。以下为完整代码：

<!DOCTYPE html>
<html>
<head>
    <title>圣杯布局</title>

    <style>
        body {
            min-width: 550px;      /* 2x LC width + RC width */
            padding:0;
            margin:0;
            text-align: center;
            font-size: 30px;
        }
        #header {
            height: 50px;
            width: 100%;
            background-color: #fa0;
        }
        #container {
            padding-left: 200px;   /* LC width */
            padding-right: 150px;  /* RC width */
        }
        #container .column {
            height: 260px; 
            position: relative;
            float: left;
        }
        #center {
            width: 100%;
            background: #D6D6D6;
        }
        #left {
            width: 200px;          /* LC width */
            right: 200px;          /* LC width */
            background: #E6D;
            margin-left: -100%;  

        }
        #right {
            width: 150px;          /* RC width */
            margin-right: -150px;  /* RC width */
            background: #77BBDD;  
        }
        #footer {
            height: 50px;
            width: 100%;
            background-color: #fa0;
            clear: both;
        }
    </style>

</head>
<body>
<div id="header">Header</div>
<div id="container">
    <div id="center" class="column">Center</div>
    <div id="left" class="column">Left</div>
    <div id="right" class="column">Right</div>
</div>
<div id="footer">Footer</div>
</body>
</html>
